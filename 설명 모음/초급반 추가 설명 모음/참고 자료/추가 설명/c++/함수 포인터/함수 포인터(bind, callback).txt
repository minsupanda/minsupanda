함수를 호출하는 과정을 fucntion call 이라 부른다.

함수 포인터(function)를 call back(나중에 부른다) / Delegate(대리자) 이라고도 한다.

Ex) 게임할 때 맵, 캐릭터, 인벤 ui를 클릭할 때 기본적으로 처리되는 과정을 대신한다?
특정좌표에 설정된 구역을 클릭시 해당 값을 보관하고 대신해서 그 값으로 후처리(해당 기능)를 한다.

함수화 느낌?
-> 그래야 나중에 스킬같은 ui를 추가할 때도 추가할 기능들만 만들어놓고 집어넣기만 하면 된다.

Bind(특정 인자를 고정)
1. 예제코드처럼은 거의 잘 쓰지 않는다
-> 애당초 전역함수를 잘 사용하지 않기 때문

2. 그럼 왜 가르쳐줬냐 -> Class 때문
-> 몬스터 Class로 몬스터 Object를 여러개 만듦
-> 몬스터 한마리에게 데미지를 주었을 때 어떤 오브젝트의 멤버함수를 호출할 지 알아야함
-> 함수포인터로 호출할때 bind를 사용해 해당 오브젝트를 함께 묶어서 보내주어 이를 해결한다
