
[UpCasting - 업캐스팅]

부모 Class를 상속받은 자식 Class가 추가될 될 때마다 증가하는 작업량을 줄이기 위해
-> 부모 Class의 포인터 변수에 자식 Class를 넣음으로써 자식 Class들을 하나의 변수로 관리하는 것

부모 포인터에 자식 Class를 저장 가능한 것 < - > 역은 성립되지 않음
=> 자식 Class가 부모 Class의 데이터를 들고있는 것이기 때문

Q. 만약 하나의 자식 Class 혼자 독립적으로 들고있는 함수를 사용하려고 한다면 업캐스팅이 사용가능한가?
A. 그럴 경우면 다운캐스팅을 하거나 (Dog포인터로 만들던가), 업캐스팅을 사용하면 안된다.
=> 나중에 배울것이기 때문에 일단 넘어감
